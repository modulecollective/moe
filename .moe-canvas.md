# Push

## PR body

Fix silent push failure: repair codex model pin, gate autopush on turn success, name agent in exit line

### Problem

`moe sdlc push` was failing silently: bureaucracy got pushed to origin
(appearing to succeed), but the moe branch never reached its remote.
The only visible signal was `claude exited: exit status 1` buried
mid-stream, followed by `committed push turn` — which looked like
success.

Root cause was a bad codex model pin (`gpt-5-mini`) that the codex CLI
rejected against a ChatGPT account with a 400. The autopush then fired
regardless, leaving bureaucracy ahead of the moe repo it was supposed
to track.

### Changes

**A. Model pin repaired** (`internal/cli/push.go`)

`pushSynthesisDefaultModel["codex"]` changed from `"gpt-5-mini"` to
`"gpt-5.4-mini"`. The previous slug is rejected by codex against a
ChatGPT account (`400 invalid_request_error`), which killed every
synthesis turn. A comment now points at `~/.codex/models_cache.json`
as the source of truth for slug selection so the next vendor rename is
a one-line edit.

**B. Autopush gated on turn success** (`internal/cli/stage.go`)

`openWikiSession`'s `closeSess` closure now takes `okToPush bool`.
`session.Close` still runs (worktree torn down, local main
fast-forwarded), but `sync.AutoPush` is suppressed when `okToPush` is
false. `runWikiSession` computes `okToPush := runErr == nil && gateErr
== nil`; `closeBootstrapFailedSession` hard-wires `false`.

This is the divergence fix — even when (A) regresses again, bureaucracy
will no longer race ahead of the project repo.

**C. Exit line names the active agent** (`internal/cli/stage.go`)

`wikiSessionInputs` gained an `Agent string`. `runStageSession`
populates it from `stageAgentName`; `runWikiSession` fills it in as a
fallback for lint/claim callers that don't pre-resolve.
`reportWikiSessionExit` prints `<agent> exited: <err>`. The hardcoded
`claude exited:` lie is gone.

### Test coverage

- `TestCloseSessSuppressesAutoPushWhenTurnFailed` — origin's

- `TestCloseSessRunsAutoPushWhenTurnSucceeded` — positive control:
  origin still tracks local main on the happy path.
- `TestReportWikiSessionExitNamesAgentInExitLine` — codex / claude /
  empty-fallback cases; codex case additionally asserts the old
  `"claude exited:"` string cannot reappear for a non-claude agent.
- `TestCloseWithAutoResolve*` — closure-signature ripple covered
  through `closeWithAutoResolve` and `closeBootstrapFailedSession`.

`go build ./...`, `go vet ./...`, `go test ./...` all clean.

### Not covered by automated tests

The live codex API interaction (model-pin returns 400) has no
automated guard — the executor seam is stubbed. The design's smoke
test (`codex exec --model gpt-5.4-mini "say pong"` → `turn.completed`)
is the contract surface. Re-run `moe sdlc push moe
shell-sources-dev-env-2026-05-15` with the rebuilt binary to validate
(A) and (B) cooperating in situ.

## Ship readiness

All three fixes land cleanly: build, vet, and the full test suite are
green, including targeted re-runs of every load-bearing case the code
canvas called out. No fixes were applied during the test stage —
everything passed on the first pass. The one remaining gap (live codex
API interaction) is a deliberate skip per the design: the executor seam

re-run the outstanding push as the final in-situ validation before
declaring done.
